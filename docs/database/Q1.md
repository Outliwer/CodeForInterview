## `SQL`编译的过程
* 解析
    * 词法解析
    
    * 语法解析
    
    * 逻辑计划
    
    * 查询优化
    
        * 在进行类似于**解释器**的分析优化，`SQL Server `对象的**架构**、**统计信息**，以及**指令的具体内容**，估算可能的执行计划，以及它们的成本，
    
    * 物理执行计划 
    
        * 在`MySql`语句完成了查询优化，将执行计划存储到`Plane Cache`，在下一次有**同样的语句执行**，可以直接在`Plane Cache`中读取物理执行计划进行执行
        
        * 对于预编译和编译的语句而言。预编译认为是不变的，因此可以进行**重用**

        
* 执行 
    * 检查用户、表权限
    * 表上加共享读锁
    * 读数据到`query cache` 
    * 取消共享读锁
